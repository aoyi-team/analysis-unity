# Unity 管理员权限警告

## 现象

Unity 启动后 Console 显示：

```
Unity is running with Administrator privileges, which is not supported.
```

## 原因

Unity Hub 默认会以管理员权限启动 Unity 编辑器。当 Windows UAC 的 `ConsentPromptBehaviorAdmin` 设为 `0`（自动提权，不弹窗）时，Hub 静默提权，Unity 检测到后报警告。

## 解决方案

修改 Unity Hub 配置，禁止其提升编辑器权限。

### 步骤

1. 确保 Unity Hub **已完全关闭**（右键托盘图标退出）

2. 打开文件资源管理器，在地址栏粘贴：
   ```
   %APPDATA%\UnityHub
   ```

3. 用记事本（或其他文本编辑器）打开 `hubConfig.json`

4. 找到这一行：
   ```json
   "hubDisableElevate":false
   ```
   改为：
   ```json
   "hubDisableElevate":true
   ```

5. 保存文件，重启 Unity Hub

> **注意**：Hub 更新后可能会重置此配置，如警告重现请重复上述步骤。

### 备选方案

如果上面方法不生效，还可以检查 Windows UAC 设置：

1. 打开 **控制面板 → 用户账户 → 更改用户账户控制设置**
2. 将滑块拉到**默认级别（从上往下第二档）**
3. 重启电脑
